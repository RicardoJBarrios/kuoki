Path

Path: string | string[]
  • Defined in packages/environment/src/lib/path/path.type.ts:15

A property name, a dot-separated set of property names or an array of property names that represents a path to get an environment value.

An environment property name must be a sequence of ASCII characters that can contain letters a-zA-Z, $, _, and digits 0-9, but may not start with a digit.

see

isPath

example
const path1: Path = 'a';
const path2: Path = 'a.b.c';
const path3: Path = ['a', 'b', 'c'];

isPath

  • isPath(value: unknown): value is Path
    • Defined in packages/environment/src/lib/path/is-path.function.ts:17

    Checks if the value is a valid path.

    A valid path is a property name, a dot-separated set of property names or an array of property names. An environment property name must be a sequence of ASCII characters that can contain letters a-zA-Z, $, _, and digits 0-9, but may not start with a digit.

    example
    isPath('a.a'); // true
    isPath('2a.a'); // false

    Parameters

    • value: unknown

      The value to check.

    Returns value is Path

    true if the value is a valid path, otherwise false.

overwritesPath

  • overwritesPath(path: Path, state: EnvironmentState): boolean
    • Defined in packages/environment/src/lib/path/overwrites-path.function.ts:23

    Checks is the path overwrites a value in the environment.

    example
    // EnvironmentState = { a: { a: {}, b: 0 } }
    overwritesPath('a.b'); // true
    overwritesPath('a.b.a'); // true
    overwritesPath('a'); // false
    overwritesPath('a.a'); // false
    overwritesPath('a.a.a'); // false

    Parameters

    • path: Path

      The path to check.

    • state: EnvironmentState

      The environment state to check.

    Returns boolean

    true if the path overwrites a value in the environment, otherwise false.
